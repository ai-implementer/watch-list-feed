from scrapbox.ioのみ時間経過でメモリ使用量が増加する原因を、Codex CLIとともに探るHelpfeel専用Cosense Export APIはMongoDB query cursorを閉じ忘れているCodex CLI.icon Helpfeel エクスポート API のコードを読み込んで、メモリが積み上がりやすい（もしくは実際にリーク扱いされる）箇所を洗い出しました。以下の点を優先的に確認・改善するのがよさそうです。カーソルを閉じていない users, events, pages それぞれで Model.find(...).cursor() を開いていますが、streamClosed でループを抜ける際に await cursor.close() していません (src/server/controllers/helpfeel/download-jsonlines.js:74-133) クライアントが途中で切断すると MongoDB 側にカーソルが残り、サーバーも最後に受け取った 4MB/101 件のバッチを抱えたまま GC 待ちになります。 Helpfeel 側から頻繁に再接続されるケースだと、カーソルとバッチが溜まり続けてメモリリーク同様の症状になります。try/finally で cursor.close() を保証してください。 mongoose 6以降ではcursor()呼び出さずにqueryを直接for await...ofで回せるを使おうshokai.icon やめた。for awaitでは明示的にcloseできないshokai.icon不要フィールドを全件読み込んでいる User.find({ _id: { $in: project.users } }) でも _id,name,email 以外のフィールド（例えばアイコンや設定値）が全部載った Document を受け取っています。projectToJsonLine も含め API では使わないデータがかなり多いため、.select({ name:1, email:1 }) と .lean() を付けるだけで 1 人あたり数 KB 〜 数十 KB 削減できます (download-jsonlines.js:70-90)。
